# frp Auth Check

批量检测 frp 服务端（frps）是否启用了 token 密码认证，并对无认证服务器进一步探测**穿透端口是否被服务端防火墙拦截**。

当 frp 服务端未配置 `auth.token` 时，任意客户端均可通过 frpc 连接并注册代理隧道，
造成严重的安全风险。本工具用于快速发现此类配置缺陷，并评估其实际可用性。

---

## 快速开始

```bash
# clone 下来直接就能用（自带 frpc 二进制）
git clone https://github.com/mcc0624/frp-auth-check.git
cd frp-auth-check

# 检测单个目标（认证检测 + 穿透可达性探测）
python3 frp_auth_check.py -t 192.168.1.100:7000

# 只测认证，跳过穿透探测（更快）
python3 frp_auth_check.py -t 192.168.1.100:7000 --no-probe

# 批量检测（默认开启穿透探测）
python3 frp_auth_check.py -l targets.txt

# 只看无需认证的（重点关注）
python3 frp_auth_check.py -l targets.txt --no-auth-only

# 输出 JSON 结果
python3 frp_auth_check.py -l targets.txt -o result.json
```

## 功能

### 1. 认证检测

使用官方 frpc 二进制连接目标 frps，按优先级尝试 4 种配置组合，判断是否需要 token 认证。

| 顺序 | 标签 | TLS | TCPMux |
|------|------|-----|--------|
| ① | 默认(TLS+Mux) | ✅ | ✅ |
| ② | 无TLS+Mux | ❌ | ✅ |
| ③ | TLS+无Mux | ✅ | ❌ |
| ④ | 纯TCP | ❌ | ❌ |

### 2. 🔥 穿透防火墙探测（新增）

对无认证的 frps 服务器，自动注册一个临时 TCP 代理，然后探测该代理端口是否可达，从而判断服务端是否配置了 iptables/firewalld 等防火墙规则拦截穿透端口。

**解决了以下场景的误判：**
```
能连 frps（端口 7000 开放）✅
但注册代理后穿透端口被防火墙拦截 ❌
→ 无密码也没用，隧道不通
```

## 输出说明

| 标识 | 含义 | 价值 |
|------|------|------|
| 🔓✅ | 无认证 + 穿透端口可达 | **最高价值**，可直接使用 |
| 🔓🔥 | 无认证 + 防火墙拦截 | 能登录但端口不通，价值有限 |
| 🔓❓ | 无认证（探测未执行） | 使用 --no-probe 跳过探测 |
| 🔒 | 需要 token 认证 | 有密码保护 |
| ❓ | 未知/端口未开放 | 不是 frp 或无法连接 |

## 检测逻辑

```
┌─ 认证检测 ──────────────┐
│  TLS/Mux 四种组合登录    │
│  → 需认证 / 无认证 / 未知 │
└──────────┬──────────────┘
           ▼  (如果是无认证)
┌─ 穿透可达性探测 ─────────┐
│ 1. 本地监听随机端口       │
│ 2. 注册 TCP 代理到 frps  │
│ 3. 尝试连接远端代理端口   │
│ 4. 能连上？→ 防火墙放行   │
│    连不上？→ 防火墙拦截   │
└─────────────────────────┘
```

## 输出格式 (JSON)

```json
{
  "host": "192.168.1.100",
  "port": 7000,
  "is_frp": true,
  "has_auth": false,
  "firewall_blocked": false,
  "status": "no_auth",
  "detail": "默认(TLS+Mux): 登录成功，无需认证 | 穿透探测: ✅ 端口 31234 可达"
}
```

新增字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `firewall_blocked` | bool/null | `false`=穿透可用, `true`=防火墙拦截, `null`=未探测 |

## 自带 frpc 二进制

仓库 `bin/` 目录预置了以下版本的 frpc，覆盖主流兼容范围：

| 文件 | 版本 | 大小 | 说明 |
|------|------|------|------|
| `bin/frpc_v0.69.1` | v0.69.1 | 17MB | 最新稳定版 |
| `bin/frpc_v0.60.0` | v0.60.0 | 14MB | 中版本兼容 |
| `bin/frpc_v0.53.0` | v0.53.0 | 14MB | 旧版本兼容 |

脚本会自动按版本从新到旧尝试，无需手动指定。

## 版本兼容性

| 服务端版本 | 协议 | 默认 TLS | 默认 TCPMux | 本工具兼容性 |
|-----------|------|---------|------------|------------|
| **v0.60 ~ 最新** | JSON | ✅ 是 | ✅ 是 | ✅ 全兼容（配置①） |
| **v0.50 ~ v0.59** | JSON | ❌ 否 | ✅ 是 | ✅ 自动降级（配置②） |
| **v0.48 及以下** | **msgpack** | ❌ 否 | ❌ 否 | ❌ 不兼容 |

## 安全提醒

- **仅测试你有权访问的服务器**
- 发现无需认证 + 穿透可达的 frp 服务器后，攻击者可任意注册代理隧道穿透内网
- 建议配置 `auth.token`：

```toml
# frps.toml
bindPort = 7000
auth.token = "your-random-secret-here"
```
